Goal orchestration for coding agents

Loop a coding agent until the goal is verifiably met.

Hand goaly a goal. It freezes what "done" means, then loops your agent against that bar until it's actually done — a bar the agent can't weaken to pass. 

$ npm i -g goaly

1 Freeze the bar

The success check is hashed and frozen before the loop starts — the agent can't weaken its own test to finish.

2 Two keys for DONE

Done means both turn: the frozen check passes and an independent, refute-first reviewer doesn't veto.

3 Fail closed

Anything that errors or returns garbage is a fail, never a false green. The control loop makes zero LLM calls.

And there's more

Everything else, one line each.

  • Best-of-N attempts--candidates N races N isolated attempts per iteration and keeps the one that climbs furthest.
  • Parallel waves--parallel-phases runs independent plan phases concurrently and re-verifies every merge.
  • Named worktrees--worktree isolates the whole run in its own checkout — your main tree is never touched.
  • Adversarial review--adversarial red-teams the contract before Seal and attacks every green before Sign-off.
  • Web UIgoaly ui is a local control center: a mission dashboard, a live run pipeline, a session inspector into the agent's turns, and the Seal in a browser review station.
  • Watch & steerTail any live run, stop cleanly, resume with a note — you can add room, never lower the bar.
  • Crash-safeA write-ahead log survives a crash at any byte; budgets survive --resume.
  • Sandboxing--sandbox jails the agent and the verify command, with per-seam network policy.
  • Follow-ups--from-run starts a new, re-verified goal that knows what just happened.
