1 Freeze the bar

The success check is hashed and frozen before the loop — the agent can't weaken its own test to finish.

2 Two keys for DONE

Done means both turn: the frozen check passes and an independent reviewer doesn't veto. The reviewer can be a multi-vote panel (--approver-quorum N) that greens only on a strict supermajority and counts any failure as a veto — and with --approver-models m1,m2,… that panel runs distinct models (reviewer i → model i) for genuine per-reviewer independence, not just variance reduction. A panel multiplies approver spend ~quorum× (metered, counted against --budget-tokens) — a small panel (≈3–5) is the recommended range, and quorum 1 (the default) is cost-neutral. --approver-lenses l1,l2,… overrides the panel's review-lens taxonomy with operator-supplied lenses (cycled when quorum > 1).

3 Fail closed

Anything that errors or returns garbage is a fail, never a false green. The control loop makes zero LLM calls.

Best-of-N parallel worker

--candidates N (alias --best-of N) runs N isolated worker attempts each iteration in linked git worktrees, scores each against the same frozen ladder, and keeps the best — the tournament is Driver-side, so the pure reducer sees exactly one winning run and never learns N existed. Candidates are graded by how far each got up the ladder — furthest up wins (an all-pass beats every partial), so two failing attempts are distinguished; the depth is read off the verdict at zero extra cost (the ladder already short-circuits at the first failing rung). Ties on depth break to lower cost, then index. On --resume a crashed fan-out re-runs only the not-yet-logged candidates — or, with --resume-best-of-incomplete collapse, collapses to the best already-logged one and re-runs nothing (fail-closed: zero logged still runs the full set). N multiplies per-iteration worker spend up to ~N× (still governed by --budget-tokens), and because each candidate is a full concurrent worker + worktree, K is capped at 16 (a value above 16 is a fail-closed error). Composes with --phased / --delta-verify / --sandbox; needs a committed HEAD.

Named worktrees

--worktree <name> re-roots the entire run — run log, lock, agent cwd, verifier, diff scope — at an isolated checkout under .goaly/worktrees/<name> on branch goaly/<name>, so the main tree is never touched and several runs can go at once. Manage them with goaly worktree create / list / remove; runs never commit, so merge-back is plain git (the end-of-run hint shows the commit + git merge goaly/<name> steps, and remove keeps the branch by default). Fail-closed throughout: removal refuses a live run inside (always) or a dirty tree (without --force). ADR 0013 →

No cheating by reimplementation

Ask goaly to build a reusable artifact and use it and a worker could satisfy a naive bar by re-deriving the logic inside the higher-level solvers and never calling the artifact — both keys pass while the thing you asked for is dead code. An independent shape classifier (a neutral call over the goal only) flags build-and-use goals, and the compiler must then author a runtime usage assertion: it spies the artifact's public entry points and asserts the verified result is produced through them — a reimplementation records zero calls and fails. A contract missing that assertion is refused at compile (COMPILE_FAILED) and re-authored with it. Fail-open, so it never blocks a non-build-and-use run.

Follow up after it ends

goaly "now also handle empty input" --from-run <id> starts a new, re-verified goal that knows what just happened: a compaction of the prior run seeds the new contract's authoring, and it runs in the same workspace. Every invariant holds — new freeze, new two-key gate. Add --inherit-session to keep the agent's memory too.

Or jump back into the CLI

goaly runs resume-cmd <id> prints how to continue the run's underlying session in its own interactive mode — claude --resume <id>, codex resume <id>, and so on. The end-of-run banner shows it too, so the next step is one copy-paste away.

The web UI

goaly ui serves a local browser view of everything the run log knows: a runs table across the workspace and every worktree (live badges included), per-run detail (the frozen contract, each iteration's verifier verdict + Sign-off, spend), and a live event feed over SSE — the write-ahead log tailed read-only, so it follows runs started in any terminal and never disturbs them. It also drives runs: start one (optionally into a worktree) through the same code path as the CLI, hold the Seal in a browser modal (approve / revise / reject — a real SealGate implementation, never a bypass), stop cooperatively, and resume with a note + raised operational caps. The Seal modal is a full review station: authored verification files render with their contents, editable in the browser or in your own editor, alongside the setup/verify commands and rubric — "re-freeze & review" re-pins the hashes into a freshly frozen contract (logged, zero LLM cost) as many rounds as you like, and only the contract you finally approve starts executing (the CLI prompt's [e]dited answer is the same mechanism). One live run per tree, worktrees for parallelism. Localhost-only, with fail-closed Host/Origin/X-Goaly-Ui guards. ADR 0014 → ADR 0015 → ADR 0016 →

Watch it & steer it

You're never locked out of a run: goaly runs watch <id> tails the write-ahead log live from any terminal (each agent turn, verdict, and veto — read-only, never disturbs the run). Ctrl-C stops cleanly between steps, then --resume <id> --note "…" hands the worker your guidance on its next turn. A run that hit an operational limit isn't a dead end — --resume with --max-iterations / --budget-tokens / --stuck-* continues it in place, auditable in the log. The extension schema simply has no field for the goal or verifier: you can add room, never lower the frozen bar. ADR 0012 →

Reliable by default

Fail closed, but never fail eagerly: a millisecond preflight catches a missing CLI / non-git workspace before any token is spent; transient 429/network/CLI blips are retried with backoff at every transport (a crashed agent turn once, LLM steps twice, Retry-After honored) while timeouts stay hard; Ctrl-C stops cleanly between steps and prints the --resume command (a second one force-exits and reaps children); the fsync'd write-ahead log survives a crash at any byte (a torn tail is repaired, a run lock stops double drivers), and --budget-tokens survives --resume. Every failed run ends with a one-line next: hint. ADR 0011 →
