Goal orchestration for coding agents

Loop a coding agent until the goal is verifiably met.

goaly runs Claude Code, Codex, Droid, pi, its own goaly-code agent against any OpenAI-compatible endpoint — or your own agent — until your goal is actually done, checked against a contract it can't weaken to pass.

1 Freeze the bar

The success check is hashed and frozen before the loop — the agent can't weaken its own test to finish.

2 Two keys for DONE

Done means both turn: the frozen check passes and an independent reviewer — prompted refute-first, cycling adversarial lenses — doesn't veto. Scale it to a multi-vote panel across distinct models →

3 Fail closed

Anything that errors or returns garbage is a fail, never a false green. The control loop makes zero LLM calls.

Best-of-N parallel worker

--candidates N runs N isolated attempts each iteration in git worktrees, scores each against the same frozen ladder, and keeps the one that climbed furthest — the pure reducer never learns N existed. Crashed fan-outs resume without re-running finished candidates.

Or just say it: "fix the flaky test, work with 4 subagents" maps onto --candidates via a deterministic grammar (never an LLM parse) and is stripped from the frozen goal. reference →

Cooperative parallel waves (experimental)

--parallel-phases runs independent plan phases concurrently as full child runs (own frozen contract, own two-key gate, isolated worktree), merges the winners with real git merge-tree plumbing, and re-verifies every merged phase on the combined tree — a merge is never trusted. Every failure shape downgrades fail-closed to the classic sequential phase, and cumulative acceptance still gates the whole run. reference →

Named worktrees

--worktree <name> re-roots the entire run — log, lock, agent cwd, verifier, diff scope — at an isolated checkout under .goaly/worktrees/<name>, so the main tree is never touched and several runs can go at once. Runs never commit; merge-back is plain git. ADR 0013 →

No cheating by reimplementation

A build-and-use goal can't be greened by re-deriving the logic inline and never calling the artifact: a neutral shape classifier flags such goals, and the compiler must author a runtime usage assertion that spies the artifact's entry points — a reimplementation records zero calls and fails. Fail-open, so it never blocks other runs.

Adversarial review (--adversarial)

Opt-in red-teaming at three points, all veto/feedback-shaped — never a third key. A critic panel attacks the authored contract before Seal (and the plan on --phased), and a refute-first panel attacks every green ladder before Sign-off — a green survives only a strict supermajority of "could not refute" votes. Off by default; a plain run is byte-for-byte unchanged. reference →

Follow up after it ends

goaly "now also handle empty input" --from-run <id> starts a new, re-verified goal that knows what just happened — a compaction of the prior run seeds the new contract's authoring. Every invariant holds: new freeze, new two-key gate. Add --inherit-session to keep the agent's memory too, or goaly runs resume-cmd <id> to jump back into the CLI session yourself.

The web UI

goaly ui serves a local view of every run — live SSE feeds, per-run detail, worktrees — and drives runs through the same code path as the CLI, holding the Seal in a browser review station: read, edit, and re-freeze the authored verification files before you approve. Localhost-only, with fail-closed request guards. reference →

Watch it & steer it

goaly runs watch <id> tails any live run read-only; Ctrl-C stops cleanly between steps; --resume <id> --note "…" hands the worker your guidance, and raised operational caps revive a run that hit a limit. The extension schema simply has no field for the goal or verifier — you can add room, never lower the frozen bar. ADR 0012 →

Reliable by default

Fail closed, but never fail eagerly: a millisecond preflight catches a missing CLI before any token is spent; transient blips are retried with backoff while timeouts stay hard; the fsync'd write-ahead log survives a crash at any byte; and budgets survive --resume. Every failed run ends with a one-line next: hint. ADR 0011 →

Sandboxed, if you ask

--sandbox jails the two untrusted execs — the agent and the verify command — via bwrap, firejail, or a container, with per-seam network policy and egress allowlists (--sandbox-net allow:<hosts>). Fail-closed: a missing mechanism refuses to start, never a silent downgrade. reference →
